# SpecStudioParser

SpecStudioParser — прототип конструктора спецификаций для nanoCAD / Model Studio CS.

Проект загружается как .NET-библиотека в nanoCAD и предоставляет Avalonia-интерфейс для работы с XML-профилями спецификаций, сканирования объектов чертежа, настройки наборов данных, фильтров, колонок и генерации табличной спецификации.

## Требования

- nanoCAD x64 25.0
- .NET 6 SDK
- Visual Studio 2022
- Доступные сборки nanoCAD в стандартной папке установки:
  - `hostmgd.dll`
  - `hostdbmgd.dll`
  - `hostMgdAvalonia.dll`

Пути к этим сборкам сейчас заданы в `SpecStudioParser.csproj` как ссылки на:

```text
C:\Program Files\Nanosoft\nanoCAD x64 25.0\bin
```

## Основные команды nanoCAD

- `SPEC_SHOW` — открыть окно конструктора спецификаций.
- `SPEC_SCAN` — выполнить быстрое сканирование активного чертежа.

## XML-профили

По умолчанию профили хранятся в папке:

```text
Документы\SpecStudioProfiles
```

В интерфейсе можно выбрать другую корневую папку, создать новый профиль, импортировать внешний XML и сохранить текущую конфигурацию.

## Текущий статус

Проект находится на стадии MVP / рабочего прототипа.

Реализовано:

- запуск Avalonia UI из nanoCAD;
- сканирование объектов ModelSpace;
- попытка извлечения параметров Model Studio CS через COM / late binding;
- загрузка базовой структуры XML-профилей;
- настройка наборов данных, типов объектов, условий и колонок;
- генерация строк спецификации с простой агрегацией;
- простая обработка формул вида `[Attribute]` и `if(...)`.

## Известные ограничения

- Движок спецификаций пока находится внутри `MainWindowViewModel` и требует последующего выноса в отдельный сервис.
- XML round-trip покрывает только базовую структуру `DatasetProfile`, `Dataset`, `Table`, `Types` и `Fields`.
- Формулы и фильтры поддерживают ограниченный набор операций.
- Сравнения `>` и `<` сейчас выполняются как строковые, а не числовые.
- Автоматических тестов пока нет.

## Рекомендуемые следующие шаги

1. Вынести расчёт спецификации в `Services/SpecificationEngine.cs`.
2. Расширить поддержку XML Model Studio без потери неизвестных секций.
3. Добавить тестовый проект для формул, XML и движка спецификаций.
4. Нормализовать имена атрибутов Model Studio CS.
